HTMLとCSSの公開について

自作したHTMLとCSSを使ったホームページを公開するにはいくつかの手順を踏む必要がある。
ここではGitHub Pagesを使った方法を中心に、公開までの作業を説明する

  1. ①ファイルを作成する
  2. ②Gitでファイルを管理する
  3. ③GitHubにファイルをアップロードする
  4. ④GitHub PagesでWeb公開する
  • 有料サーバを使う場合

①ファイルを作成する

まず、自分のパソコン上でホームページを構成するファイルを作成する。

  • index.html：ホームページの本体
  • style.css：ページのデザインを記述するスタイルシート

※今回はDreamweaverを使って、実際にコードを書きながら作成している。

②Gitでファイルを管理する

作成したファイルを「変更履歴付き」で管理するために、Gitというツールを用いる。
作業フォルダ内でGitBashを使って以下のコマンドを実行する：

git init：Git管理を開始する

git add .：すべてのファイルをステージング

git commit -m "初回コミット"：初期状態として履歴に保存

その後も修正のたびに、以下のように段階的にコミットしていく。

git add .

git commit -m "自己紹介セクションを追加"

③GitHubにファイルをアップロードする

GitHub上にリポジトリを作成する。
GitBashで以下のようにGitHubと接続し、ファイルをアップロードする：

git remote add origin https://github.com/ユーザー名/リポジトリ名.git

git branch -M main：mainブランチに切り替え

git push -u origin main：GitHubに反映

④GitHub Pagesでホームページとして公開する

GitHubには「Pages」という機能があり、アップロードしたHTMLをホームページとしてそのまま公開できる。

リポジトリの「Settings」→「Pages」に移動し、

  • Branch：main
  • Folder：/rootを選択することでホームページがWeb上で公開される

数十秒〜1分でURLが発行され、世界中からアクセス可能になる。

有料サーバを使う場合

GitHub Pagesは、静的ファイル（HTML/CSS/JS）をそのままブラウザに配信する仕組みのため、サーバー側で動くプログラム（バックエンド処理）は一切動作しない。
たとえば、ログイン処理や投稿・検索機能などを備えたページを作るには、サーバーでPHPやPythonなどを動かす環境が必要になりGitHub Pagesはこれに対応しない。

①サーバ契約

さくらのレンタルサーバ、Xserverなどでサーバを契約する。

②独自ドメインを取得する

独自ドメインを取得して自分だけのURLを使用できる。
ただし、多くのレンタルサーバでは無料で使えるサブドメインも用意されているため、独自ドメインがなくても問題なく公開できる。

③ファイルをアップロード

作成したHTMLやCSSファイルをレンタルサーバに送信する。FTPソフト(ファイル転送ソフト)を使って送信する。
ファイルは、レンタルサーバ内のpublic_htmlフォルダなどに入れることで、ブラウザからアクセスできるようになる。

④Web公開完了

正しくアップロードされていれば、ブラウザでURLにアクセスすることでページが表示される。
これで、GitHub Pagesを使わない方法でもHTML/CSSを公開できる。
